How Django's Default Template Engine Works

A template in Django is a text file, usually an HTML file, that defines the static structure of a web page and contains special syntax for inserting dynamic content. Django uses the Django Template Language (DTL) as its default template engine. DTL allows developers to write templates using variables, filters, and template tags to control how data is displayed.

When a template is rendered, Django parses and compiles it into an internal representation. This representation may be cached in memory for performance reasons. The template engine then combines the template with context data provided by the view and produces a final HTML string, which is returned to the client as part of an HTTP response.

Template Tags

Block Tag

The block tag is used to support template inheritance and serves different roles depending on where it is defined.

  • In a base (parent) template, a block acts as a placeholder that child templates can override.
  • In a child template, a block provides content that replaces the corresponding block in the base template.

Loops: for ... in

The for tag is used to iterate over sequences such as lists, dictionaries, or querysets.

Keywords

  • reversed: iterates over the sequence in reverse order.
  • empty: defines a block that is rendered if the sequence is empty or does not exist.

Loop Variables

  • forloop.counter: current iteration, starting at 1.
  • forloop.counter0: current iteration, starting at 0.
  • forloop.first: true if this is the first iteration.
  • forloop.last: true if this is the last iteration.
  • forloop.revcounter: iterations remaining, starting at 1.
  • forloop.revcounter0: iterations remaining, starting at 0.
  • forloop.length: total number of items in the loop.
  • forloop.parentloop: reference to the surrounding loop in nested loops.

If Control Structures

If Statement

  • The if tag evaluates a condition and renders a block of content if the condition is true.

Elif

  • The elif tag is evaluated if previous conditions are false.

Else

  • The else tag renders content when none of the preceding conditions are met.

Operators

Django template if tags support a limited set of logical and comparison operators used to evaluate conditions.

  • and / or: combine multiple conditions.
  • == / !=: test equality or inequality.
  • < > <= >=: compare numeric or comparable values.
  • in / not in: test membership in a list, dictionary, or queryset.
  • is / is not: test identity or boolean values such as None.

Display of Passing Context Variables

Context variables are passed from the view to the template as a dictionary, where each key represents a variable name and each value represents the corresponding data. This context is provided to the template when rendering the response using the render() function.

Inside the template, context variables are accessed and displayed using double curly braces ({{ variable_name }}). During rendering, the template engine replaces these variables with their actual values in the final HTML output.
